# Tuition Track Project Documentation

This document provides a comprehensive overview of the Tuition Track project, breaking down the architecture, directory structure, and the purpose of each individual file within the workspace.

## 1. Project Overview
Tuition Track is a full-stack web application designed for managing tuition centers, students, teachers, classes, attendance, and payments. 
- **Backend:** Built with [Bun](https://bun.sh/), [Hono](https://hono.dev/), and [Prisma ORM](https://www.prisma.io/) with an SQLite database.
- **Frontend:** A Single Page Application (SPA) built with [React](https://react.dev/), [Vite](https://vitejs.dev/), and [Tailwind CSS](https://tailwindcss.com/).

The workspace is organized into a monorepo-style structure with two main directories: `backend/` and `frontend/`.

---

## 2. Backend Architecture (`/backend`)

The backend follows a layered architecture pattern: **Routes → Validators → Services → DB**. This separates concerns, making the code more maintainable and testable.

### Root & Configuration Files
- `package.json` / `bun.lock`: Define the project dependencies (Hono, Prisma, Zod, etc.) and run scripts (e.g., `bun run dev`).
- `tsconfig.json`: TypeScript compiler configuration for the backend.
- `prisma.config.ts`: Configuration for Prisma setup.
- `.env`: Environment variables (e.g., Database URL, JWT Secret, Port).

### Database & Prisma (`/backend/prisma`)
- `schema.prisma`: The single source of truth for the database schema. It defines models for `User`, `Student`, `Teacher`, `Class`, `Enrollment`, `Attendance`, `Payment`, and `Notification` and maps them to SQLite tables.
- `seed.ts`: A script used to populate the database with initial/dummy data for testing and development.
- `dev.db`: The local SQLite database file where the data is actually stored.
- `migrations/`: Contains SQL files generated by Prisma representing schema changes over time.

### Source Code (`/backend/src`)

#### Entry Point
- `index.ts`: The main application entry point. It initializes the Hono app, sets up global middleware (CORS, Logger), registers all API routes, and configures the global error handler.

#### Database Connection (`/src/db`)
- `index.ts`: Initializes and exports the Prisma Client instance, allowing the rest of the app to communicate with the SQLite database.

#### API Routes (`/src/routes`)
These files map HTTP methods and endpoints to specific service functions, often applying middleware and validation first.
- `auth.routes.ts`: Endpoints for login, registration, and session management.
- `student.routes.ts`, `teacher.routes.ts`, `class.routes.ts`: CRUD operations for core entities.
- `attendance.routes.ts`: Endpoints for recording and retrieving attendance records.
- `enrollment.routes.ts`: Endpoints to handle enrolling students into classes.
- `payment.routes.ts`: Endpoints for processing and tracking fee payments.
- `dashboard.routes.ts`: Aggregated endpoints that provide summary statistics for the frontend dashboard.
- `notification.routes.ts`: Endpoints for fetching user notifications.

#### Services (`/src/services`)
This is the core "Business Logic" layer. Services contain the actual logic to fetch data, manipulate it, and apply business rules before responding.
- `auth.service.ts`: Handles user authentication, password verification, and JWT token generation.
- `student.service.ts`, `teacher.service.ts`, `class.service.ts`: Handle database queries (via Prisma) for fetching and mutating entity records.
- `attendance.service.ts`, `enrollment.service.ts`, `payment.service.ts`: Logic specific to these domain operations.
- `dashboard.service.ts`: Logic to aggregate and calculate dashboard metrics.
- `notification.service.ts`: Logic to dispatch and fetch notifications.

#### Validators (`/src/validators`)
Defines [Zod](https://zod.dev/) schemas to ensure incoming request data (bodies, parameters, queries) is strictly typed and formatted correctly before hitting the services.
- `auth.validator.ts`, `student.validator.ts`, `class.validator.ts`, `payment.validator.ts`, `attendance.validator.ts`: Provide schemas for their respective domains.

#### Middleware (`/src/middleware`)
Functions that intercept incoming requests to perform checks or modifications.
- `auth.middleware.ts`: Validates JWT tokens in the `Authorization` header and attaches the decoded user to the Hono context.
- `rbac.middleware.ts`: Role-Based Access Control. Ensures a user has the required role (e.g., `ADMIN` or `TEACHER`) to access a specific route.
- `error.middleware.ts`: Centralized error handling that catches exceptions (like Zod validation errors or custom AppErrors) and formats them into standardized JSON responses.

#### Utilities (`/src/utils`)
Helper functions used across the application.
- `errors.ts`: Defines custom error classes (like `AppError`) to standardize HTTP error responses.
- `jwt.ts`: Helper functions to sign and verify JSON Web Tokens securely.
- `password.ts`: Utility for hashing passwords (e.g., using bcrypt) and comparing plain text passwords against hashes.

#### Types (`/src/types`)
- `index.ts`: Contains custom TypeScript interfaces and types shared across the backend to ensure type safety.

## 4. Frontend Architecture (`/frontend`)

The frontend is a React application utilizing Vite as the bundler. It uses Tailwind CSS for styling and React Router for client-side navigation.

### Root & Configuration Files
- `package.json` / `bun.lock`: Defines frontend dependencies (React, React Router, Tailwind, Lucide Icons, React Hook Form, etc.).
- `vite.config.ts`: Configuration for Vite, including React plugins and absolute path resolution.
- `tsconfig.json`, `tsconfig.app.json`, `tsconfig.node.json`: TypeScript configurations for the Vite environment.
- `eslint.config.js`: Configuration for code linting.
- `index.html`: The root HTML file that mounts the React application.

### Source Code (`/frontend/src`)

#### Entry Points
- `main.tsx`: The absolute entry point that renders the `<App />` component into the DOM `root` element.
- `App.tsx`: The root React component. It sets up the `BrowserRouter`, defines the application's routing (Public vs. Protected routes), and wraps the application in context providers (Auth, Error Boundary).
- `index.css`, `App.css`: Global CSS files where Tailwind directives (`@tailwind base`, `@tailwind components`, `@tailwind utilities`) and custom global styles are defined.

#### API Integration (`/src/api`)
- `client.ts`: An Axios instance pre-configured with the backend base URL (e.g., `http://localhost:3000/api`) and interceptors for handling auth tokens and global API errors.

#### Components (`/src/components`)
Reusable, isolated React components.

**UI Components (`/src/components/ui`)**
These are generic, stateless, atomic design elements styled with Tailwind.
- `Button.tsx`: A highly reusable button component with variants (primary, secondary, danger).
- `Card.tsx`: A container component for displaying information in a card format.
- `Input.tsx`: A styled form input field, often integrated with React Hook Form.
- `Label.tsx`: A styled label for form inputs.
- `Table.tsx`: A reusable data table component with styled headers and rows.

**Layout Components (`/src/components/layout`)**
Components that define the structure of the pages.
- `AppLayout.tsx`: The main layout wrapper for authenticated pages. It typically includes the `Sidebar` and a main content area.
- `AuthLayout.tsx`: A simplified layout wrapper specifically for the login/registration pages (usually a centered card).
- `Sidebar.tsx`: The main navigation menu displayed on the left side of the dashboard, adjusting links based on the user's role.

**Error Handling**
- `GlobalErrorBoundary.tsx`: A wrapper component that catches unhandled JavaScript errors anywhere in the component tree and displays a fallback "Something went wrong" UI instead of crashing the whole app.

#### Contexts (`/src/contexts`)
- `AuthContext.tsx`: Uses React Context to provide global state management for the currently authenticated user. It exposes functions like `login`, `logout`, and the `user` object to any component that needs it.

#### Utilities (`/src/lib`)
- `utils.ts`: Contains generic helper functions for the frontend. Most commonly includes functions like `cn` (combining `clsx` and `tailwind-merge`) to conditionally merge Tailwind CSS classes dynamically.

#### Pages (`/src/pages`)
These are the top-level View components that correspond directly to React Router paths. They orchestrate data fetching and render layout/UI components.
- `Login.tsx`: The authentication page containing the login form.
- `Dashboard.tsx`: The main landing page after login, displaying high-level metrics, recent activities, or charts.
- `Students.tsx`: The page dedicated to listing, adding, and managing students.
- *(Placeholders exist in App.tsx for future pages like `Teachers.tsx`, `Classes.tsx`, `Attendance.tsx`, and `Payments.tsx`)*.
